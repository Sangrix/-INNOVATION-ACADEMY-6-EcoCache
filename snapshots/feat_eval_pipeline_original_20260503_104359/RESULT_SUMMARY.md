# CIASC alpha/threshold 수정 후 로컬 GPU 평가 결과

- 실행일: 2026-05-03
- 실행 환경: Windows, NVIDIA GeForce RTX 3050 4GB Laptop GPU
- 브랜치: `fead/fix-ciasc-alpha-threshold`
- 탄소 집약도 설정: `CIASC_FIXED_CI=430`, `CARBON_INTENSITY_G_PER_KWH=430`
- 평가 문항: 50개
- 평가 방식: `B1`, `B2`, `CIASC alpha=0.25`, `CIASC alpha=0.50`, `CIASC alpha=1.00`

## 수정 사항

이번 실행은 기존 `feat/eval-pipeline` 브랜치의 평가 흐름을 기준으로 하되, CIASC 계산부에서 확인된 문제를 수정한 뒤 다시 측정한 결과입니다.

- `alpha`가 내부에서 `0.15`로 고정되던 문제를 제거하고, 실행 옵션의 `alpha` 값이 실제 임계값 계산에 반영되도록 수정했습니다.
- CIASC 임계값이 이전 질의의 결과를 기준으로 누적 증가하던 문제를 제거하고, 매 질의마다 고정 기준값에서 다시 계산되도록 수정했습니다.
- `run_eval.py`의 CO2 집계가 `metrics`와 `carbon_metrics` 키 차이 때문에 0으로 나오는 문제를 보완했습니다.

## 임계값 설정

탄소 집약도는 고정 비교를 위해 `430 gCO2/kWh`로 두었습니다. 수정 후 CIASC 임계값은 질의마다 누적되지 않고 동일하게 유지됩니다.

| 설정 | 임계값 |
| --- | ---: |
| B1 | 1.1000 |
| B2 | 0.9000 |
| CIASC alpha=0.25 | 0.7417 |
| CIASC alpha=0.50 | 0.7333 |
| CIASC alpha=1.00 | 0.7167 |

## 지연 시간 특이 사항

`CIASC alpha=0.25`의 평균 지연이 높게 나온 이유는 전체가 느렸기 때문이라기보다 특정 문항 하나가 크게 튄 영향입니다.

- `TQ-020`: `608,771 ms`, 약 10분 8초
- 해당 값을 포함한 평균 지연: `26,346 ms`
- 해당 값을 제외한 보정 평균 지연: 약 `14,460 ms`

실행 중 노트북 화면 꺼짐, 절전 전환, 백그라운드 작업, 전원 상태 변화 등이 발생하면 wall-clock latency가 크게 튈 수 있습니다. 따라서 `alpha=0.25`의 평균 지연은 논문 표에 그대로 사용하기보다 재측정하거나 outlier 처리 기준을 명시하는 것이 안전합니다.

## alpha=1.00의 CO2가 높게 나온 이유

`alpha=1.00`은 캐시 히트율이 `32.0%`로 가장 높았지만, 총 CO2는 `alpha=0.50`보다 높게 나왔습니다. 로그 기준으로 보면 캐시 히트율 차이는 1문항 수준인 반면, 실행 중 GPU/노트북 전력 상태가 더 높게 잡힌 영향이 큽니다.

| 설정 | documents 평균 CO2 |
| --- | ---: |
| CIASC alpha=0.50 | 약 21.75 mg |
| CIASC alpha=1.00 | 약 27.30 mg |

즉 `alpha=1.00`은 캐시를 조금 더 맞췄지만, 해당 실행 구간에서 GPU 평균 전력 또는 노트북 전체 전력 상태가 더 높아 총 CO2가 증가한 것으로 해석됩니다. 평가 중 다른 작업을 같이 보거나, 화면/전원/팬/온도 상태가 달라진 경우에도 이런 차이가 발생할 수 있습니다.

## alpha=1.00 전력 상태 보정 CO2

원본 측정값은 그대로 보존하되, 전력 상태 차이를 줄여 비교하기 위해 `alpha=0.50` 실행의 안정적인 경로별 평균 CO2를 기준으로 `alpha=1.00`을 보정해볼 수 있습니다.

- 보정 기준: `alpha=0.50`
- QA cache hit 평균 CO2: 약 `10.90 mg`
- documents retrieval 평균 CO2: 약 `21.75 mg`
- `alpha=1.00` 경로 수: QA cache hit `16건`, documents retrieval `34건`

계산식은 다음과 같습니다.

```text
alpha=1.00 normalized CO2
= 16 * 10.90mg + 34 * 21.75mg
= 913.9mg
= 0.9139g
```

| 설정 | Raw CO2 | 전력 상태 보정 CO2 | 해석 |
| --- | ---: | ---: | --- |
| CIASC alpha=1.00 | 1.1227 g | 약 0.9139 g | 실행 중 전력 상태 상승분을 제거하면 alpha=0.50보다 약간 낮은 수준 |

보정값 기준으로 보면 `alpha=1.00`은 B2의 `1.1268 g` 대비 약 `18.9%` CO2 감소로 해석할 수 있습니다. 다만 이 값은 실제 측정값을 대체하는 값이 아니라, 노트북 실행 상태 차이를 줄여 알고리즘 경로 차이를 비교하기 위한 보조 지표입니다.

## 결론

이번 실행 기준으로 가장 균형이 좋은 설정은 `CIASC alpha=0.50`입니다.

- B2 대비 정확도: `68.0% -> 92.0%`
- B2 대비 캐시 히트율: `2.0% -> 30.0%`
- B2 대비 평균 지연: `17,435 ms -> 15,060 ms`
- B2 대비 총 CO2: `1.1268 g -> 0.9246 g`

다만 CO2는 GPU 전력 측정값과 탄소 집약도 계수를 곱해 계산한 추정치이므로, 논문용 최종 표에는 동일 조건에서 반복 측정한 평균값을 사용하는 것이 좋습니다.
