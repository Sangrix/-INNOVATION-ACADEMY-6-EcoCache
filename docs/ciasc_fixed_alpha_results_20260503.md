# CIASC alpha/threshold 수정 및 로컬 GPU CO2 평가 정리

## 배경

기존 `feat/eval-pipeline` 브랜치의 CIASC 실행 로그를 확인했을 때, `alpha=0.25`, `0.50`, `1.00` 설정이 서로 다르게 동작해야 하는데도 임계값 변화 폭이 동일하게 보이는 문제가 있었습니다. 확인 결과 CIASC 계산부에서 `alpha`가 내부적으로 `0.15`로 고정되어 있었고, 임계값도 질의마다 이전 값 위에 다시 계산되면서 누적되는 문제가 있었습니다.

이번 브랜치에서는 기존 평가 파이프라인의 구조는 유지하면서, CIASC 계산이 실행 옵션의 `alpha`와 고정된 탄소 집약도 조건을 제대로 반영하도록 수정했습니다.

## 수정한 내용

- `carbon_optimizer.py`에서 `alpha=0.15` 고정값을 제거했습니다.
- `run_eval.py`에서 전달한 `alpha` 값이 CIASC 임계값 계산에 실제로 들어가도록 수정했습니다.
- 임계값이 이전 질의 결과를 기준으로 누적되지 않도록, 매 질의마다 기준 임계값에서 다시 계산되도록 수정했습니다.
- CO2 집계가 0으로 나오는 문제를 막기 위해 `metrics`와 `carbon_metrics` 키를 모두 읽을 수 있게 보완했습니다.

- GPU: NVIDIA GeForce RTX 3050 4GB Laptop GPU
- 문항 수: 50문항
- 탄소 집약도: `430 gCO2/kWh` 고정


## 임계값 계산 결과

`CIASC_FIXED_CI=430` 조건에서 수정 후 계산된 임계값은 다음과 같습니다.

| 설정 | 임계값 |
| --- | ---: |
| B1 | 1.1000 |
| B2 | 0.9000 |
| CIASC alpha=0.25 | 0.7417 |
| CIASC alpha=0.50 | 0.7333 |
| CIASC alpha=1.00 | 0.7167 |

이제 `alpha` 값이 커질수록 임계값이 더 낮아지고, 그 결과 캐시 히트율이 증가하는 방향으로 동작합니다.

| --- | ---: | ---: | ---: | ---: | ---: |

## 결과 해석

이번 결과에서 가장 균형이 좋은 설정은 `CIASC alpha=0.50`입니다. B2와 비교했을 때 정확도는 `68.0%`에서 `92.0%`로 올라갔고, 캐시 히트율은 `2.0%`에서 `30.0%`로 증가했습니다. 동시에 총 CO2는 `1.1268 g`에서 `0.9246 g`으로 줄어 약 `17.9%` 감소했습니다.

`CIASC alpha=0.25`는 평균 지연이 `26,346 ms`로 높게 나왔지만, 로그를 보면 `TQ-020` 하나가 `608,771 ms`, 즉 약 10분 8초로 크게 튄 것이 원인입니다. 이 문항 하나를 제외하면 평균 지연은 약 `14,460 ms` 수준으로 내려갑니다. 따라서 이 값은 알고리즘 자체가 느려졌다기보다 실행 중 노트북 화면 꺼짐, 절전 상태, 백그라운드 작업, 전원 상태 변화 같은 외부 요인이 섞였을 가능성이 큽니다.

`CIASC alpha=1.00`은 캐시 히트율이 `32.0%`로 가장 높았지만, 총 CO2는 `1.1227 g`으로 `alpha=0.50`보다 높게 나왔습니다. 실제 로그 기준으로 `documents` 검색 단계의 평균 CO2를 보면 `alpha=0.50`은 약 `21.75 mg`, `alpha=1.00`은 약 `27.30 mg`입니다. 즉 캐시를 한 문항 더 맞췄지만, 해당 실행 구간의 GPU/노트북 전력 상태가 더 높게 측정되어 총 CO2가 올라간 것으로 볼 수 있습니다.

평가 중 다른 작업을 같이 보거나, 화면 밝기/전원 모드/발열/팬 속도/백그라운드 프로세스가 달라지면 같은 코드라도 CO2 값이 흔들릴 수 있습니다. 그래서 최종 논문 표에는 가능하면 동일 조건에서 반복 측정한 평균값을 사용하는 것이 좋습니다.

## 요약

- 기존 문제였던 `alpha=0.15` 고정과 임계값 누적 문제를 수정했습니다.
- `CI=430` 조건에서 `alpha`가 커질수록 임계값이 낮아지고 캐시 히트율이 증가하는 흐름을 확인했습니다.
- 이번 단일 실행 기준 대표 결과는 `CIASC alpha=0.50`이 가장 안정적입니다.
- CO2 측정값은 실제 GPU 전력 측정 기반이지만, 노트북 환경에서는 실행 중 상태 변화에 민감하므로 반복 측정이 필요합니다.
